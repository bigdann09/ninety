# Ninety — Design cleanup, feature fixes, and new integrations

## Context
Ninety is a live on-chain football prediction market built on Solana/Anchor.
Dark green base (#0B1410), orange accent (#F97316 approx), teal used sparingly.
The core design language is teletext-tight — high information density, monospace
numerals, minimal decoration. Every change below must preserve that identity.
Do not introduce new accent colors. Do not add gradients, glows, or decorative
panels unless specifically asked below.

The project is being submitted to a hackathon. Judges will see this. Any UI that
reads as "AI generated boilerplate" or has inconsistent accent colors is actively
harmful to the submission — treat cleanup as critical path, not cosmetic.

---

## Priority 0: Settlement pipeline integrity check — do this before anything else

Before touching UI, verify the settlement pipeline is writing
`settlement_tx_signature` correctly for every settled stake. This field
is the source of truth for everything downstream — receipts, NFT metadata,
explorer links. If it's null on any settled stake record, stop and fix the
settlement worker before proceeding.

To verify: place a stake on devnet, trigger or wait for settlement, query
the StakeAccount PDA and confirm `settlement_tx_signature` is populated with
a real base58 signature. Click it manually in Solana Explorer devnet before
writing a single line of receipt UI. A beautiful UI pointing to a dead or
missing transaction is worse than no UI at all.

---

## Priority 1: NFT Receipt Stubs — real minted NFTs on Solana

This is the most important judged feature. Every judge from TxODDS will
click "Verify on Solana" first. It must go somewhere real — a real minted
NFT on devnet with real on-chain metadata.

### Minting approach — Metaplex Core (not legacy Token Metadata)

Use **Metaplex Core** (`@metaplex-foundation/mpl-core`) — it's the current
standard, cheapest per-asset on Solana, and has the cleanest API for
minting single assets without the Token Metadata program overhead.

Install:
```bash
npm install @metaplex-foundation/mpl-core @metaplex-foundation/umi
npm install @metaplex-foundation/umi-bundle-defaults
npm install @metaplex-foundation/umi-uploader-irys

When to mint
Mint exactly once, triggered server-side immediately after settle_market
is confirmed on-chain. Do not mint client-side — the keeper process that
runs settlement owns the mint CPI. This means:
Settlement worker confirms settle_market transaction
Worker calls mint function with settlement data
NFT is minted to the user's wallet address
Mint address + metadata URI stored in your Postgres stakes table
(nft_mint_address, nft_metadata_uri columns — add these migrations)
Frontend reads these fields and surfaces the NFT in the receipts tab

Minting code — Metaplex Core via UMI

import { createUmi } from '@metaplex-foundation/umi-bundle-defaults';
import { mplCore, create } from '@metaplex-foundation/mpl-core';
import { irysUploader } from '@metaplex-foundation/umi-uploader-irys';
import { keypairIdentity, generateSigner, publicKey } from '@metaplex-foundation/umi';

async function mintReceiptNFT(stake: SettledStake): Promise<string> {
  const umi = createUmi(process.env.RPC_URL)
    .use(mplCore())
    .use(irysUploader())
    .use(keypairIdentity(keeperKeypair)); // your existing keeper keypair

  // 1. Generate SVG and upload metadata
  const svg = generateReceiptSVG(buildSVGData(stake));
  const imageUri = await umi.uploader.uploadJson({ svg }); // or upload as file

  const metadata = buildReceiptMetadata(stake, imageUri);
  const metadataUri = await umi.uploader.uploadJson(metadata);

  // 2. Mint the NFT to the user's wallet
  const asset = generateSigner(umi);
  await create(umi, {
    asset,
    name: `Ninety Receipt · ${stake.homeTeam} vs ${stake.awayTeam}`,
    uri: metadataUri,
    owner: publicKey(stake.userWallet), // minted directly to user
  }).sendAndConfirm(umi);

  return asset.publicKey.toString(); // this is the mint address
}

Call mintReceiptNFT from your settlement worker immediately after
settle_market confirms. Store asset.publicKey as nft_mint_address
in Postgres.

Fix the "1th minute" ordinal bug with this helper:

function ordinal(n: number): string {
  const s = ['th','st','nd','rd'];
  const v = n % 100;
  return n + (s[(v - 20) % 10] || s[v] || s[0]);
}
// ordinal(1) → "1st", ordinal(2) → "2nd", ordinal(43) → "43rd"

  'Chile': 'cl', 'Iran': 'ir', 'South Korea': 'kr',
  'Real Madrid': null, 'Arsenal': null, // club teams: no flag
  // extend as needed

function FlagImg({ team }: { team: string }) {
  const iso2 = TEAM_FLAGS[team];
  if (!iso2) return null;
  return (
    <img
      src={`https://flagcdn.com/20x15/${iso2}.png`}
      width={20} height={15}
      alt={team}
      style={{ display: 'inline', verticalAlign: 'middle', marginRight: 4 }}
    />
  );
}
